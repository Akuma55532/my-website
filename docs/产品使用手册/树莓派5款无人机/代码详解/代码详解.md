# 树莓派 5 端代码详解

树莓派端由 `detect.py` 负责相机、识别、跟随控制和视频发送，`autodeploy.sh` 负责一次性部署。运行入口为 `autostart.sh`。

```text
Picamera2 -> YOLO 检测 -> 速度指令 -> utils.datalink_serial.datalink -> 飞控
                 |
                 +-> 标注图像 -> JPEG/UDP -> 192.168.0.110:5005
```

## `detect.py`

### 职责与启动流程

1. 用 Picamera2 以 `640×480`、`RGB888` 格式采集画面。
2. 加载 `yolo26n_ncnn_model`，仅检测类别 `0`（通常为人），置信度阈值为 `0.5`。
3. 根据检测框位置生成飞行速度，并调用 `datalink.set_pose()` 发送给飞控。
4. 将带检测框的画面压缩成 JPEG，通过 UDP 发送。
5. 按 `q` 退出窗口循环。

### 识别与跟随

`YOLO26Detector.detect()` 使用 `self.model(..., task="detect")`，因此它是逐帧检测加视觉跟随，不是带 ID 的 YOLO 多目标追踪。

| 配置 | 默认值 | 作用 |
| --- | ---: | --- |
| `TrackDown` | `False` | `False`：前视模式；`True`：俯视模式。 |
| `k_forward` | 1.2 | 前后速度比例系数。 |
| `k_side` | 0.35 | 俯视横移比例系数。 |
| `k_yaw` | 0.4 | 前视偏航比例系数。 |
| `max_forward` | 0.6 | 前后速度限幅。 |
| `max_side` | 0.4 | 横移速度限幅。 |
| `max_yaw` | 0.35 | 偏航速度限幅。 |
| `target_width_ratio` | 0.4 | 前视时目标框宽度的期望画面占比。 |

前视模式根据目标框宽度控制前后距离、根据水平偏差转向：

```python
cmd_linear_x = clamp(k_forward * (target_width_ratio - width_ratio), max_forward)
cmd_angular_z = clamp(k_yaw * error_x, max_yaw)
```

目标框小于期望值时前进，超过期望值时后退；目标偏离画面中心时通过偏航修正。俯视模式不转向，而是根据目标在画面中的上下、左右偏差进行前后和横移：

```python
cmd_linear_x = clamp(-0.3 * k_forward * error_y, max_forward)
cmd_linear_y = clamp(0.3 * k_side * error_x, max_side)
```

当前实现只取检测结果的第一个框。多人同时出现时，控制目标可能在不同人之间切换；未检测到目标时不会调用 `set_pose()`。

### UDP 视频流

`TranstreamThread` 用独立守护线程发送视频，队列最多保存 5 帧。队列满时丢弃最旧帧，优先保证低延迟。

图像先按 `scale_percent` 缩放，再按 `frame_quality`（JPEG 质量，默认 70）压缩。每个 UDP 数据报为：

```text
UV01 | 头长度 | 序号 | 秒 | 纳秒 | 宽 | 高 | 通道数 | JPEG质量 | 载荷长度 | JPEG数据
```

头部格式为 `!4sHIIIHHBBI`：网络字节序，总长度 28 字节。单个数据报最大允许 65507 字节，超过时该帧被丢弃。

### 通信模块接口

`detect.py` 依赖 `utils.datalink_serial` 中的 `datalink`：

```python
datalink().set_pose(linear_x, linear_y, linear_z, angular_z)
```

启动时还会分别在线程中运行 `datalink.drone` 和 `datalink.heartbeat`。当前工作区的 `utils` 目录为空，缺少该模块源码；应补入实际通信文件后，再补充串口配置、报文格式、接收循环和心跳机制的逐函数说明。

## `autodeploy.sh`

该脚本用于首次部署，执行顺序如下：

1. 修改 `/boot/firmware/config.txt`，插入 IMX219 相机覆盖配置并关闭自动相机识别。
2. 更新系统软件包，安装 `python3-libcamera` 和 `python3-picamera2`。
3. 在项目目录创建 `.venv`，且通过 `--system-site-packages` 复用系统安装的相机库。
4. 将 pip 源设置为中科大镜像，升级 pip。
5. 安装仓库内的 `sympy-1.14.0.tar.gz`，并安装 `ultralytics[export]`。
6. 询问是否立即重启，使相机配置生效。

脚本使用 `set -e`，任一步骤失败都会立即退出。它会修改系统相机配置、更新系统包和写入全局 pip 配置，应只在树莓派首次部署或需要重建环境时执行。

## `autostart.sh`

脚本激活 `/home/pi/yolo/.venv` 后执行 `python detect.py`。它假定项目部署目录固定为 `/home/pi/yolo`，并且当前工作目录能够找到 `detect.py`。
